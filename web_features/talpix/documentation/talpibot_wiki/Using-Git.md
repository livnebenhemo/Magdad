אין מה להילחץ, בפעם הראשונה הסקריפט ההתחלתי עושה את זה למענכם.

# תהליך כתיבת קוד
הגישה שלנו מבוססת על [העמוד הבא](https://guides.github.com/introduction/flow/) מבית גיטאהב.

> **אמ"לק** - מה שיש בMaster חייב לעבוד היטב ולהיות מוגמר, והוא מה שירוץ בסוף על המכונה שלנו. 

אם אתם מעוניינים לעבוד על משהו בפרוייקט, לדוגמה לתקן באג, להוסיף יכולת, לשנות מבנה וכו' כל מה שאתם צריכים לעשות הוא לעבור על הנקודות הבאות:

### ליצור בראנצ'
צרו ענף חדש בגיט שבו תקדמו את היכולת. תנו לו שם משמעותי. כדי ליצור ענף חדש כתבו את הפקודה הבאה:

```bash
$ git checkout -b feature-vidutz-fixing-display-bug"

-->   Switched to a new branch "feature-vidutz-fixing-display-bug"
```

#### שם לבראנצ'

כדי למנוע בלאגן, יש שמות מסודרים לבראנצ'ים. תיצרו בראנצ' לפי החוקיות הבאה:
```תחילית-<שם פיצ'ר>``` לדוגמא ```feature-test```

| תחילית | משמעות |
| ------ | ------ |
| feature- | בראנץ' שמייצג עבודה על פיצ'ר מסוים |
| bugfix- | בראנץ' שמייצג תיקון באג מסוים | 
| technology- | בראנץ' שמייצג עבודה על תשתית טכנולוגית של הבוט |

### כתבו את הקוד שלכם
כעת כשאתם בענף החדש, אתם יכולים לקדם את הקוד שלכם. הוסיפו commits חדשים עם כל התקדמות שלכם (שוב - **שמות משמעותיים!**).


**חשוב**: מדי פעם תורידו שינויים חדשים מ`branch` ה`master` לתוך ה`branch` שלכם. כדי לעשות זאת ב`PyCharm`:
```bash
VCS -> Git -> Merge Changes -> Remote/Origin/master
```

### סיום
כעת לאחר שסיימתם, הגיע הזמן לשלב בחזרה לmaster. זהו רגע מרגש, שבו הקוד שלכם סוף סוף עולה לאוויר.

לפני שאתם עושים זאת, חשוב לשלב לענף שלכם את העדכונים משאר הצוות. כדי לעשות זאת, **להשלים**
